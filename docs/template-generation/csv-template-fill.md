# CSV/Table Template Fill Flow

Use this flow when a finished InDesign source page must be copied many times and only labeled text fields change. The CSV may remain on Linux: Python reads each cell and sends the exact string to the Mac MCP. The model chooses the configuration and mapping once; it does not copy row values into prompts or issue hand-written per-row edits.

## Choose The Correct Page Tool

`create_derivative_page` and `duplicate_template_page` have different purposes:

| Tool | Use | Copy behavior |
| --- | --- | --- |
| `create_derivative_page` | Creative generated layouts | Creates a new page and optionally copies labeled editable motifs. It is not a complete source-page duplicate. |
| `duplicate_template_page` | Finished templates with text-only row changes | Duplicates the complete source page through InDesign, preserving its existing images, placed graphics, shapes, styles, swatches, layers, and geometry as normal InDesign duplication allows. It patches copied slot labels for derivative targeting and rejects duplicate `derivativeId` values by default. A fallback item-by-item duplication path is not implemented yet. |

Do not use `create_derivative_page` as if it copies every object from a finished template page.

## Source Template Requirements

Work only in a template workspace working copy. The source page must contain labeled text frames for every field that Python will update. A recommended source label is:

```json
{
  "slot": "name",
  "role": "title",
  "editable": true
}
```

`duplicate_template_page` merges the new `derivativeId` into each copied slot label while preserving `slot`, `role`, `editable`, and other source metadata. For example, the copied slot becomes targetable with:

```json
{
  "derivativeId": "invite_001",
  "slot": "name"
}
```

Slot names must be unique on the copied page across all labeled copied items by default. An ambiguous duplicate slot is an error, not a best-effort update.

## Recommended Flow

1. Inspect the workspace and validate that the active document is `work/current.indd`.
2. Inspect the source page and confirm that every configured slot exists once.
3. Export a cheap checkpoint preview of the source page when visual fidelity matters.
4. Run `duplicate_template_page` once per selected CSV row with a stable `derivativeId`.
5. Run `update_text_slot` for each mapped field using `labelQuery: { derivativeId, slot }`.
6. Optionally inspect the derivative, run checks, and export checkpoint previews. For large batches, sample previews unless the user requests every page.
7. Save the working copy or a version.

Treat the returned `pageIndex` as current evidence only. Use `derivativeId` for subsequent derivative-scoped work.

## Linux Runner

The included runner uses Python's standard library and does not copy the CSV to the Mac:

```bash
python scripts/fill_template_from_csv.py \
  --csv examples/template_rows.csv \
  --config examples/template_fill_config.json \
  --out fill_result.json
```

Use `--dry-run` first to validate headers, mappings, row IDs, and generated derivative IDs without connecting to MCP or mutating InDesign. `--offset` and `--limit` can restrict a smoke test. `--collect-errors` continues after row/slot failures; the default is fail-fast. `--no-save` overrides `saveWorkingCopy` for a smoke test. `--save-on-error` is an explicit unsafe override that allows saving even when errors were recorded. `--export-preview` opts into checkpoint preview export.

The runner reads CSV with `encoding="utf-8-sig"` and `newline=""`. It does not strip or normalize cells. Quoted commas, Hebrew, empty optional cells, and other exact UTF-8 values are passed directly as `update_text_slot.text`.

## Configuration

```json
{
  "mcpUrl": "http://mac-hostname-or-ip:3333/mcp",
  "sourcePageIndex": 0,
  "rowIdColumn": "id",
  "derivativeIdPattern": "invite_{id}",
  "slotColumnMap": {
    "name": "name",
    "title": "title"
  },
  "saveWorkingCopy": true,
  "exportPreview": false,
  "limit": null,
  "offset": 0
}
```

- `sourcePageIndex` identifies the finished source page only at duplication time.
- `derivativeIdPattern` uses Python field substitution against each row. `invite_{id}` and row ID `001` produce `invite_001`.
- `slotColumnMap` maps InDesign slot names to CSV column names. Here, slot `title` receives the exact `title` column value.
- `saveWorkingCopy` saves once after the selected rows finish.
- `exportPreview` defaults off to avoid expensive previews for every row.
- `limit` and `offset` select a deterministic row range.
- `fitSlots` is optional. Set it to an explicit slot-name list, or `true` for all mapped slots, only when separate post-update fitting is intended. Fitting is off by default.

One update generated by the runner has this shape:

```json
{
  "labelQuery": {
    "derivativeId": "invite_001",
    "slot": "name"
  },
  "text": "exact CSV cell value",
  "preserveStyle": true,
  "textReplacePolicy": "isolatedOnly"
}
```

The runner never sends `fit: true` to `update_text_slot`.

## Failure And Result Reporting

Before any InDesign mutation, the runner validates required config keys, CSV headers, non-empty row IDs, safe generated derivative IDs, and uniqueness of all selected derivative IDs. It then checks `/health` and `/bridge-status`, initializes one Streamable HTTP MCP session, opens the working copy, and validates the active document. A validation response with `ok:false` now stops the run before any page duplication.

`fill_result.json` records row index, row ID, derivative ID, page identity, and each slot's source column, UTF-8 SHA-256, source length, returned object identity, warnings, and errors. Failures include `rowIndex`, `rowId`, `slot`, `column`, `tool`, and `error` where available. If `saveWorkingCopy` is enabled but any row/slot/tool error occurred, the runner skips saving by default and records `saveSkipped: true` with `saveSkippedReason: "errors_present"`.

Claim batch completion only when the result reports all selected rows processed and no row/slot errors. This proves deterministic transfer and successful tool calls, not visual correctness. When the final document matters, inspect/export previews and copied link/slot evidence. For large batches, preview samples or checkpoints unless every preview was requested.

## Safety Rules

- Never mutate the original `.indd`; use the template workspace working copy.
- Do not use UI automation, text edit mode, selection, or the active page as targeting state.
- Do not use arbitrary filesystem access or `execute_indesign_code`.
- Do not carry raw `pageIndex` across mutation steps when `derivativeId` exists.
- Do not call `update_text_slot` with `fit:true`; fit separately after inspection if explicitly configured.
- Do not silently continue when copied slot names are ambiguous.
- Do not save a partially failed batch unless `--save-on-error` is explicitly set.
- Do not treat threaded/shared/raw duplicated text as normal editable text. `duplicate_template_page` reports text diagnostics, and `update_text_slot` with `isolatedOnly` refuses unsafe stories.

## Live Validation Checklist

The handler and runner have local static/unit coverage. Full-page InDesign duplication remains unverified until a live Mac/InDesign/UXP pass proves it.

Use a source page containing:

- one placed image frame with a valid link
- one background shape
- two labeled, styled text slots
- one unlabeled decorative item

Then duplicate the page, update both slots from the example CSV, export a checkpoint preview, inspect copied slots and links, and confirm that the image, background, styles, geometry, and unlabeled decorative item remain visible. Record any threaded/overset diagnostics and do not claim live validation until this passes through the real bridge/plugin/InDesign session.
